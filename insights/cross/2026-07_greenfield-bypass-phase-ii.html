      核心论点：前作把阶段 Ⅱ 定义为"决胜关卡" —— 谁愿意跳进 J 曲线的谷底，谁就能捞到阶段 Ⅲ 红利。但历史上真正走通到阶段 Ⅲ 的那家公司，用的不是这个策略。福特 1913 年建高地公园工厂时，整个阶段 Ⅱ 才刚刚开始 —— 他没等 J 曲线走完，也没痛苦地改造旧车间，他直接在一块空地上从零建了一个 greenfield，把电气化 + 流水线 + 单元驱动一起装进去。这条路径的现代版就是 OPC / 一人公司。这篇是我对前作留下的问题 —— "know-how 会不会消失" —— 的自我作答。
      这篇文章想回答的三个问题
      1. Greenfield 是不是真的绕过了阶段 Ⅱ 的谷底？ —— 用福特 1913 的具体动作反推。
      2. 如果从 OPC 起步、行业 know-how 全没了，会不会出大问题？ —— 用 Polanyi 的显性/暗知识二分回答。
      3. 哪些行业能这么玩、哪些不能？ —— 用 tacit 密度作为分层判据。

      一阶段 Ⅱ 有三条路径 · 视频没讲第三条#

      Three forks at the entrance · 拆车间 / 装电机不拆 / Greenfield

      前作里我把阶段 Ⅱ 描述成一个二选一：要么装电机不拆车间（rent dissipation 吃光收益）、要么拆车间做先驱（跳进 J 曲线谷底、忍受 5-15 年负 ROI）。但这个二分不完整 —— 它默认了"车间已经存在"这个前提。

      如果根本没车间要拆呢？这就是第三条路：Greenfield。历史上福特就是这么做的 —— 他不是把底特律某家马车厂"升级"过来的，是在一块空地上从零建 greenfield。老对手（Studebaker、Packard）都在痛苦地改造旧车间；福特直接跳过阶段 Ⅱ 的谷底，第一天就把电气化 + 流水线 + 单元驱动一起装进去。

      Three Forks · At the Entrance of Phase II
      Path A装电机不拆 · 卡在阶段 Ⅱ 入口
      Path B拆车间先驱 · 前作论述的 GE/Anthropic 路线
      Path CGreenfield · 本文论点 · 福特/OPC 路线

      福特路线的关键在"没车间可拆"。1913 年高地公园工厂建成时，产量 5 年内从年产 2 万 → 年产 100 万，单车工时从 12.5 小时 → 93 分钟。这不是他"熬过"了 J 曲线，是他从来没有进入 J 曲线。

      「拆一家旧车间的成本，永远比在空地上新建一家还高 —— 因为你要同时支付：既得利益补偿、沉没资产折旧、老员工再培训、以及最贵的那一项 —— 组织记忆的路径依赖。」 —— 我对 Acemoglu-Restrepo 2026 QJE rent dissipation 论文的口语化转述

      放到 AI 时代 —— 传统企业的"车间"不是物理设备，是已有的组织架构、KPI 体系、审批流程、部门壁垒、沉没的 SaaS 合同。这些东西在阶段 Ⅱ 全部变成待拆的旧车间。而 OPC / 一人公司 / 极小团队 从第一天就没有这些东西，所以它们不用跳进谷底 —— 它们直接从阶段 Ⅲ 的车间形态起步。

      二Know-How 会消失吗？—— 不是消失，是迁移#

      Polanyi 定理帮我们回答 · Codified 归 LLM · Tacit 归个人+Agent

      这是我对 greenfield 路径最初的焦虑：如果一家公司从 OPC 起步，那几十年积累的行业 know-how —— 客户关系、销售话术、监管信任、供应链谈判技巧、"这个客户就是不喜欢周三开会"—— 会不会全没了？如果全没了，OPC 是不是根本立不起来？

      这个焦虑得拆成两半才好答。Michael Polanyi 1966 年在《The Tacit Dimension》里做了一个至今仍然成立的二分：codified knowledge（显性知识，能写成文档的）和 tacit knowledge（暗知识，只能通过身体经验传递的）。这两类 know-how 在 LLM 时代面临的命运完全不同。

      ① Codified · 显性知识
      能写下来的 · 已被 LLM 吃掉大半
      • SOP / best practices / API 文档
      • 行业报告 / 教科书 / 法条
      • 标准销售话术 / 客服 FAQ
      • 软件/法律/会计/营销大部分知识
      ✅ 已经吃掉大半 · 对 OPC 不是问题，反而是红利
      ② Tacit · 暗知识
      写不下来的 · 只能人带过来
      • 老销售看客户脸色的直觉
      • 外科医生的手感 / 匠人的火候
      • B2B 关系资本 / 供应链信任
      • "这个客户就是不喜欢周三开会"
      ❌ LLM 吃不掉 · 对 OPC 是关键约束 · 必须由人带过来

      结论比"know-how 消失"精确得多：know-how 没有消失，是载体在迁移。

      • Codified know-how 的载体：从"组织的规章制度 / 老员工的脑子" → "LLM 权重 + 你的 Agent 栈"
      • Tacit know-how 的载体：从"组织" → "个人 + 他的 Agent"

      所以 OPC 起步不等于从零 know-how 起步 —— 它等于由持有 tacit know-how 的那一个人，配上一整个 Agent 栈。这是关键。福特招的也不是刚出学校的孩子，招的是有金属加工经验的老工人，只是让他们用新方式工作。"人的经验保留 + 组织的路径依赖丢弃" —— 这才是 greenfield 战术的完整公式。

      Polanyi 定理 · 为什么 tacit 一定要人带

      Michael Polanyi 1966 年提出："We can know more than we can tell."（我们知道的永远比我们能说清的多。）也就是说，所有可以完全表达出来的知识都是 codified 的，剩下那部分—— tacit —— 定义上就不能被完全 codified。这就是 LLM 时代 tacit know-how 的天花板：不管模型多强，只要它的输入是文本/图像/视频，它就永远只能吃掉能被记录下来的那部分。剩下那部分只在人的身体里。

      Polanyi · The Tacit Dimension · 1966

      三哪些行业能 OPC 起步 · Tacit 密度光谱#

      Where OPC works · Tacit density is the axis

      既然 tacit know-how 是 OPC 的关键约束，那判断一个行业能不能从 greenfield 起步，就只需要问一个问题：这个行业的 tacit 密度有多高？

      Tacit-Density Spectrum · OPC Viability
      中间层是 FDE 派工模式的靶点：你把甲方大脑里的 tacit know-how 抽取出来，装进 OPC 或极小团队 + Agent 栈，替代传统公司整个部门。这是 前作 §三 Unit Drive 那节的深层逻辑，也是 Palantir FDE 模式的存在理由。

      三层的判据不是"这个行业有多难"，是"把这个行业的关键 know-how 说清楚要多久"。

      • 低 tacit（OPC 直接起飞）：软件、内容、设计、简单法律咨询、简单会计、Marketing 自动化、独立开发者产品。已经在跑 —— Cursor / Midjourney / Cognition / v0 / Perplexity 都是样本，ARR/员工比拉爆传统 SaaS 一个量级。
      • 中 tacit（OPC + FDE 下沉）：垂直 SaaS、行业数据分析、专业咨询、企业 IT 服务。这是我最看好的切入位置。tacit 存在但不高到穿透不了 —— FDE 派工的价值就在这里：把甲方大脑里的暗知识抽取到 Agent 栈。
      • 高 tacit（OPC 短期做不了）：医疗、金融合规、复杂 B2B 销售、跨境供应链、监管密集行业。关系资本 + 监管信任无法用 Agent 复制，至少这一波不行。
      分层的边界会随时间移动

      今天的"高 tacit"不见得永远高。1980 年代律师工作里的判例检索是重度 tacit（老律师的直觉记忆），2000 年 Westlaw / LexisNexis 把它 codified 了一半，2024 年 Harvey / CoCounsel 把它再 codified 一层。技术每往前走一步，就有一批 tacit 变成 codified，OPC 的适用边界就往右移一段。这条边界不是永远，是滚动的锋线。

      四1913 → 2026 · 福特路线的现代对应#

      Ford 1913 High-land Park → 2026 OPC / one-person company

      把福特 1913 年的具体动作和 2026 年 OPC 的具体动作摆在一起对照，会看到结构上几乎一比一映射：

      Ford High-land Park · 1913
      在空地上建工厂
      阶段 Ⅱ 才走 1/3 · 电机采纳率仅 5-25%

      没车间要拆 —— 集中电机 + 单元驱动 + 流水线一次装齐；老工人从马车厂招过来（tacit 保留），但工作方式全新（组织重构）。

      ≡ 1913 → 2026 100 年 · 同一结构
      OPC · One-Person Company · 2026
      在空账户上建公司
      阶段 Ⅱ 才走 1/3 · 企业 AI 采纳率 < 20%

      没组织要拆 —— LLM + Agent 栈 + 端到端工作流一次装齐；founder 带着 tacit 从上家公司出来（人的经验保留），但组织形态全新（一人 = 一家公司）。

      这个映射不是修辞。它逐条对应：

      要素Ford 1913OPC 2026
      物理边界 底特律外围一块空地 一个空账户 · 一台电脑
      动力源 大电机 + 独立单元驱动 Foundation Model + Agent 编排层
      作业单元 每台机器一个岗位 · 单一工序 每个 Agent 一个 skill · 单一职能
      连接方式 流水线传送带 Prompt chain / Workflow / MCP
      Tacit 保留 老工人从马车厂招过来 · 只是工作方式重排 Founder 从上家公司出来 · 只是组织方式重排
      没有的东西 旧车间 · 旧工会规则 · 沉没资产 旧组织 · 旧 KPI · 旧 SaaS 合同 · 旧部门壁垒

      历史押韵到这个程度，我很难不严肃对待。

      五作为 FDE · 我怎么在这条路径上定位#

      Where I stand · 在 Greenfield 的哪一层押注

      把上面三段论证放在一起，作为 FDE，我给自己画一条清晰的活动带：

      我的三条判断
      1. 低 tacit 层不是我的战场：Cursor / Perplexity 这类已经卷成红海，OPC 直接起飞不需要 FDE 介入。
      2. 中 tacit 层是 FDE 的红利窗口：垂直 SaaS / 行业数据分析 / 专业咨询 —— 我要做的是把甲方大脑里的 tacit know-how 抽取到 Agent 栈。这就是 Palantir FDE 模式的通用化。
      3. 高 tacit 层观察就好：医疗、金融合规、跨境供应链短期还是传统组织的堡垒。等 tacit → codified 的锋线滚动过来再说，不着急。

      换句话说：我不是要做拆车间的先驱（前作 Path B） —— 那需要在传统巨头内部有权力位置、能忍受 5-15 年的负 ROI。我要做的是 greenfield 的施工方（本文 Path C） —— 帮已经决定不进 J 曲线的团队，把新车间一次建对。这是 FDE 派工模式的现代版：不改造存量、只服务增量。

      「阶段 Ⅲ 的赢家可能不是从阶段 Ⅱ 的先驱里长出来的，而是从阶段 Ⅰ 结束之后、直接建 greenfield 的人里长出来的。」 —— 本文核心论点

      六待验证的问题#

      1. OPC 的规模天花板在哪里？ —— 福特 1913 是从零建的，但到 1929 年高地公园有 7 万工人。OPC 会不会在增长到某一规模时被迫开始"装电机不拆车间"？还是说 Agent 栈让规模上限无限展开？
      2. Tacit 边界的滚动速度 —— Harvey / CoCounsel 把法律 tacit codify 用了 3 年。医疗、金融合规、跨境供应链的 tacit 需要多久？这条边界一年往右移多少行业？
      3. Founder 的 tacit 供给是不是新瓶颈 —— 如果 OPC 的关键是"由持有 tacit 的那一个人"起步，那 tacit 密集行业里"敢从大厂出来单干"的人会不会成为稀缺资源？这是不是一种新的"人才军备竞赛"？
      4. Path B 和 Path C 的对撞 —— 传统巨头里的 Path B 先驱（比如 Anthropic 内部改造）和 Path C 的 greenfield 施工方（比如你/我）会在哪里首次直接竞争？我的直觉是垂直 SaaS 市场，但没有证据。
