      建议优先押注
      Outcome·Continuity·World
      核心结论：2024–2026 的 Agent 是 Artifact——你打开它、它交付一个产物、你关闭它，按 seat/token 付费。2027–2028 的范式是 Presence——你订阅一个结果、它持续在场、跨主体缔约，按 outcome 付费。由此衍生五种存在形态：意图订阅体、认知连续性体、世界模型体、A2A 社会、感官网格体。关键判断：OS/Factory 仍是 2026–2027 的基础设施（runtime、连接器、治理），五种形态是产品与社会层的跃迁；Factory 若进化，应从「造 App」转向「孵化 outcome 契约 + world schema + 连续性包」。
      一页要点
      • 范式：Artifact（打开→任务→关闭）→ Presence（订阅→事件→批准）；付费从 seat/token 到 outcome
      • 五形态：意图订阅体（订结果）·认知连续性体（第二自我）·世界模型体（活地图）·A2A 社会（Agent 缔约）·感官网格体（事件驱动在场）
      • 成熟度：多数仍 ★～★★ 试点/早期；2028 跃迁依赖 契约/记忆/世界模型/信誉/传感 五类基础设施成熟
      • 组合：五形态非互斥——ToB 套餐 = 结果订阅 + 连续性 + 世界切片，聊天只是入口之一
      • FDE 押注：2027 前优先 Outcome + Continuity + World（单切片）；A2A/Mesh 跟协议与硬件成熟再进

      一背景：从「多一个聊天窗口」说起#

      The Shift · 为什么需要新形态

      企业今天买 Agent，多半是「多了一个聊天窗口」——报表多了，KPI 未必动；按人头、按 token 越用越亏，不敢承诺效果。这条路的天花板是：用户买的不是对话，是结果。一旦结果可定义、可验证、可定价，Agent 的存在形态就会从「你打开的工具」变成「持续替你盯到底的在场面」。

      由此延伸出一个问题：如果 OS/Factory（怎么造、怎么管 Agent）是地基，那么 2027–2028 客户真正感受到的产品长什么样、方案怎么卖？答案是五种「在场」形态——它们不是更聪明的对话框，而是把结果契约、认知连续性、世界模型、Agent 间缔约、传感触发分别做成可订阅的存在。

      二核心范式：Artifact → Presence#

      Paradigm Shift · 单位、交互、付费、壁垒全变

      理解五形态之前，先锁定范式转移本身。2024–2026 的 Agent 围绕 Artifact（产物）组织：构建部署一个 Agent 应用，用对话或 Workflow 驱动，按租户 RBAC 管控。2027–2028 围绕 Presence（在场）组织：订阅一个结果，事件触发持续行动，跨主体用信誉缔约。

      2024–2026 · Artifact

      单位是应用/模板，交互是打开→任务→关闭，壁垒在连接器 + playbook。

      → 范式转移
      2027–2028 · Presence

      单位是委托关系/认知体/世界切片，交互是订阅→事件→批准，壁垒在连续性 + 世界模型 + 信誉。

      四个维度的变化值得逐条记住——它决定了未来的产品怎么设计、怎么收费、护城河在哪：

      维度Artifact（OS/Factory）Presence（五种形态）

      付费Seat / TokenOutcome / saved hours / 缔约手续费

      三五种形态全景#

      Five Forms · 各占一个「在场」维度

      Presence 不是单一产品，而是五个正交维度的存在形态。它们共享「持续在场 + 事件驱动」的底座，但各自押注不同的基础设施：结果契约、认知主权、世界模型、Agent 间协议、传感网格。

      五个节点围绕 Presence，各押注一类基础设施。当前多数处于 ★～★★ 试点/早期，2028 跃迁窗口在 2027–2028。

      逐个形态

      1

      意图订阅体Outcome Subscriber

      比喻：保洁按干净付钱——客户买的是「事办成了」，不是「聊得开心」。

      定义
      用户声明可验证的结果契约，系统负责分解、租用子 Agent、调度工具、在 checkpoint 请求人类授权，按结果达成计费或担责。
      现状种子
      Anthropic Outcomes API（rubric + 独立 grader + 自纠正）、Managed Agents、OpenAI Operator（cron）、Workspace Agents、process-based pricing（$1.50/工单）、ARS 支付担保。
      关键差距
      契约语言缺长期 KPI；清算与仲裁无标准；与采购合同/AP2 未衔接；授权链缺分级委托宪法。
      实现路径
      Contract DSL（KPI/截止/例外/数据源）→ 分解器 → 第三方 Grader 市场 → Ledger（达成率/退款/AP2）→ Gate（P0 自动 / P1 审批 / P2 禁止）。

      2028 跃迁结果定价 + 可执行契约 + 清算仲裁 · 观察信号头部云发 Outcome Marketplace、采购合同出现 Agent SLA 条款、独立 grader 成为 SKU。

      2

      认知连续性体Continuity Agent

      比喻：换机数据跟着走——经验跟着岗位走，不锁死在某个工具里。

      定义
      跨 session、产品、模型商，持续维护用户/角色的认知状态（偏好、决策史、文风、失败教训），形成可授权、可导出、可继承的「第二自我」。
      现状种子
      Anthropic Memory Import（ChatGPT/Gemini JSON → Claude）、Onoma 跨厂商 memory API、Dreaming（批处理 1–100 sessions 可审核）、CLAUDE.md/Scope 项目连续性。
      关键差距
      无 Continuity Pack 统一格式；导入 ≠ 细粒度分享；缺组织 vs 个人双层宪法；进化缺回归测试。
      实现路径
      Continuity Pack（身份 + episodic/semantic/procedural + 元认知，签名版本化）→ 睡眠流水线（去重/矛盾检测/提炼 skill）→ 晋升闸门（P0/P1/P2）→ 可移植导入导出 → 组织交接（认知体副本 + 范围限制 + 审计）。

      2028 跃迁用户主权记忆 + 授权继承 + 组织认知交接 · 观察信号开放 Continuity Pack 标准、企业岗位认知体 SKU、Dreaming 成默认后台。

      3

      世界模型体World-model Agent

      比喻：导航心里有路——脑子里有张业务实况表，而不是每次重新查接口。

      定义
      维护可推理的活世界切片（实体、关系、状态、因果），API 主要用于校准真值与执行副作用。企业侧即「订单—库存—运力—客户」联合状态。
      现状种子
      Traversal Production World Model（遥测 + 因果，百万实体）、DigitalTwinz 数字孪生、Geminos KnowledgeWay 知识图谱、ODEI 宪法化 ontology；主流仍是 RAG + Tool + ERP（每次重拉）。
      关键差距
      构建成本高、冷启动需 FDE；图推理 vs LLM 规划缺标准接口；ERP/传感器/人工录入需 truth maintenance；个人生活世界模型几乎空白。
      实现路径
      业务事件流 → 摄入对齐 → 世界图（行业 ontology + 状态机）→ 模拟（what-if）→ 图上规划的 Agent → 阈值触发 API 校准写回。落地顺序：单切片 → 最小 ontology → 只建议不写 → 模拟与跨切片。

      2028 跃迁事件驱动状态机 + 因果模拟 + 行业 ontology · 观察信号World Graph 与 Agent Studio 并列 SKU、招标从 RAG 转向「状态机 + 模拟」。

      4

      A2A 社会Agent-to-Agent Ecology

      比喻：比价下单你只付款——机器先谈，人只批大事。

      定义
      主路径是 Agent ↔ Agent（发现、委托、谈判、履约、结算）；人类是立宪者、仲裁者、法律责任承担者。协议：A2A task + AP2 mandate + 信誉分。
      现状种子
      Google A2A v1.0（Agent Card + Task，150+ 组织）、AP2 v0.2（Intent/Cart/Payment Mandate）、MCP（Agent↔工具）、Copilot/Bedrock/LangGraph 宣称支持。
      关键差距
      谈判语义缺还价/要约标准；无跨平台信誉系统；责任链（平台/部署方/模型方/用户谁赔）未定；需联邦世界模型；国内支付路径不同。
      实现路径
      Agent Card 发现 → A2A Task + Artifact 交换 → RFQ/Quote 商务扩展 → AP2 Mandate 支付链 → 信誉 registry（完成率/争议率）→ 内网网关联邦。国内可走飞书/钉钉/微信支付宝托管代扣，不必等全球协议。

      2028 跃迁信誉 + 支付 + 责任链产品化 · 观察信号A2A negotiation/contract 扩展、Agent 错误责任险 SKU、Agent 店铺与 Card 搜索引擎。

      5

      感官网格体Sensory Mesh Agent

      比喻：烟感响了才报警——有事才找你，聊天框降为批准台。

      定义
      以多模态传感事件流为主输入，阈值触发主动行动；对话 UI 降为批准台。
      现状种子
      ContextAgent（NeurIPS 2025，可穿戴→主动调工具）、ProAgent（AR 眼镜 + 边缘，+33% 主动预测）、Meta Ray-Ban 等硬件；企业 IoT 多无 LLM Agent 层；Operator cron/Workspace 定时属弱主动。
      关键差距
      连续录音录像的隐私合规（个保法/GDPR）；缺 interruption budget 打扰成本；端云实时 multimodal 贵；主动建议难对赌 SLA。
      实现路径
      传感网格 → 边缘脱敏（敏感数据不出端）→ 事件总线（统一 UserContextEvent）→ 打扰策略（价值 × 紧急 − 打扰成本，每日配额）→ 轻量情境图 → 批准 UX（卡片「将要做 X，因为 Y」）→ MCP/A2A 执行。

      2028 跃迁可穿戴 + 阈值策略 + 批准式 UI · 观察信号OS 级 Agent 通知类别、可穿戴第三方 Agent SDK、连续感知 AI 法规。

      五组合：五形态不是互斥的#

      Composition · ToB 套餐怎么搭

      客户不会单买某个形态，而是买组合。典型 ToB 套餐把两三个形态捆成一个可验收、可复用的方案——聊天只是入口之一，真正卖的是「能盯 KPI、能沉淀 SOP、能接实况数据的一整套」。

      结果订阅 + 评测

      可验收 AI 劳力

      客服「一次解决」、财务「对账本周清零」、合同「高风险 100% 标出」。今天就能卖：Rubric + 独立评测 + 合同验收附件。

      报价：试点 seat + 按闭环加价

      第二大脑 + 审批

      岗位知识资产

      销售（客户偏好）、客服（标准答）、法务（红线）、研发（架构约定）。记忆归客户、可导出 = 招标加分项。

      报价：按部门记忆容量

      活地图 + 告警感应

      运营实况助手

      供应链（在途 + 异常）、制造（订单—产能—物料）、零售（门店库存—补货）。先只读看板 → 再建议人批 → 再模拟。

      报价：按系统数 + 事件量

      两种完整方案故事，说明组合如何落地：

      • 制造 · 订单与交付：活地图（订单—物料—产能）→ 结果订阅（准时率）→ 第二大脑（插单规矩）→ 感应（产线异常推卡片）→ 成熟后再对外要货 Agent。
      • 零售 · 门店与补货：活地图（库存 + 在途）→ 感应（低于安全库存推店长）→ 结果订阅（缺货率）→ 第二大脑（促销话术规矩）。

      底下都要 Agent OS：权限、日志、评测、接 ERP/工单——这是地基。行业在变的五条信号：招标写验收标准附件、出现按闭环定价、客户问记忆归谁、合同写 AI 误操作怎么赔、产品默认推送批准而非打开聊天。

      六对 FDE / 产品路线的含义#

      Field Mapping · 押注、动作、现有资产

      1. i优先 Outcome + Continuity + World（单切片）：2027 前这三条最可落地、最能把「省时」变成「可验收 impact」。Outcome 走 rubric + 独立 grader；Continuity 做 skill/memory 晋升规范与 Pack 导出；World 从单切片 entity + FSM 起步。
      2. iiA2A 跟协议、Mesh 跟硬件：A2A/AP2 仍欧美中心、Mesh 受限于可穿戴与隐私法规——这两条先跟踪、做内网网关/Computer Use harness 的小 POC，不重仓。
      3. iiiFactory 进化方向：从「造 App」转向「孵化 outcome 契约 + world schema + 连续性包」。现有资产（顺丰 KPI、sf_claw 进化、skill 评测、MCP 目录、物流 WMS 事件）正好分别对应五形态的种子。
      4. iv销售话术升级：不再卖更聪明的对话框，卖「能盯 KPI、能沉淀 SOP、能接实况数据」的一整套；报价从 seat/token 转向按闭环/按结果。
      5. v不必五线全开：判断成熟度而非热度——哪个形态的基础设施（契约/记忆/世界模型/信誉/传感）先就绪，哪个就先可产品化。
      判断

      2028 的 Agent 不是更强的 Artifact，而是持续在场的 Presence——你订阅结果、它盯到底、跨主体缔约。对 FDE，真正的机会不在「选哪个形态」，而在把 Outcome + Continuity + World 三条基础设施先做厚，让组合套餐可验收、可复用、可定价；A2A 与 Mesh 等协议和硬件成熟再进。
