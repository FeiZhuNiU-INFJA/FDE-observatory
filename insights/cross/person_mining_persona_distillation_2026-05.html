      从数字痕迹蒸馏人

      Person Mining 作为底层能力与品类

      2026-05-21 调研产出 品类洞察 ~6,000 字 时机：进行中 信心：中

      与已有框架的关系：横切 2028 Agent 五种形态 的「第二大脑」形态、衔接 认知影响力 的「认知透明」方向。

      「从一个人的全量数字痕迹中蒸馏出可计算的人物表示」是一个底层能力，不是一个产品。它可以衍生出至少 15 种上层应用，从离职交接、退休专家方法论提取，到临终数字 Agent、创作者 AI 副本。业界目前已经分裂成 5 个流派，但没有任何一家同时握住底层能力和应用层——这是当前最大的结构性空位。
      一页结论
      • 不要做通用「人格 SDK」——大模型厂商和学术界两头夹击，正在把它做成 commodity
      • 不要做创作者 AI 副本——Delphi.ai 已经赢家通吃
      • 不要做临终 Agent——$125B 成熟产业 + 伦理黑洞
      • 推荐 做 ToB 离职交接 / 退休知识捕获——海外有验证、国内空白、当下变现
      • 推荐 做高客单的专家方法论提取（律师 / 医生 / 资深工程师 / 分析师）——市场零散无寡头
      • 长期 评测 + 认知透明审计——所有现有玩家都没做

      真正的产品核心差异化是 Active Inquiry（主动追问）——不是被动 RAG，而是 AI 在试图复现决策时主动识别盲区并向当事人追问。这是把咨询型产品升级成 SaaS 的关键，也是竞品都没做透的地方。

      一核心洞察#

      1.1 底层是同一件事

      离职交接 Agent、临终数字 Agent、创作者 AI 替身、个人第二大脑、退休专家方法论提取、客户洞察镜像——这些看似不同的产品，底层都是同一个能力：

      定义

      Person Mining / Persona Distillation：把一个人的非结构化数字痕迹（邮件、聊天、文档、代码、声音、视频）压缩为可计算的人物表示（知识图谱、SOP、风格指纹、人格向量、可对话 Agent）。

      1.2 但产品层会分叉

      维度工程难点的位置
      评估标准「对不对」（决策复现）vs「像不像」（情感载体）vs「商业化好不好」（创作者经济）
      数据访问企业合同已授权 vs 死者无法同意 vs 本人主动授权
      错误代价业务损失 vs 情感伤害 vs 法律风险 vs 商业声誉
      时间维度当下用爆 vs 跨 30 年使用 vs 持续运营

      底层共享 ~60-70%，产品 / 工程共享 ~30-40%。"同一底层能搞 N 个产品"是过头乐观，但"沉淀底层、专攻一两个应用、横向扩展"是真实可行的策略。

      1.3 为什么是现在

      三个并行的技术红利在 2026 合流：

      红利一

      长上下文 + RAG 成熟

      处理一个人全量数据从天文成本降到几百块。

      红利二

      激活空间人格抽取

      2026 多篇 arXiv 证明：人格是 LLM 表征空间里近似正交的方向向量，可做向量代数，不需要 fine-tune。

      红利三

      Memory 中间件成熟

      Mem0 / Letta / Zep 提供了基础设施，不用从零造轮子。

      成本 / 精度的跃迁让以前只能玩票的应用变成可商用。

      关键判断

      评测的空白是这个赛道里最被低估的子方向——没有评测，企业不敢付钱。

      2.6 Active Inquiry：从被动挖掘到主动追问（关键差异化）

      前面所有处理步骤都是被动挖掘——只能拿到已被表达过的知识。但 42-60% 的隐性知识根本没被写下来，因为：

      • 员工自己觉得"理所当然"，意识不到这是知识
      • HR 不知道该问什么（没有当事人语境）
      • 继任者不知道自己不知道什么（unknown unknowns）
      核心论点

      只有 AI 在试图复现一个决策时，才会发现"这里有个空白"——这是任何静态文档化都做不到的，也是被动挖掘的根本上限。

      这把产品形态从「一次性」变成「持续运行」

      维度静态版（被动挖掘）动态版（Active Inquiry）
      形态离职前 4-6 次会谈 → 输出文档离职前 30/60/90 天就部署的影子学徒
      商业模式一次性咨询，¥10-50k/人持续运行 SaaS + 月费
      客户感知"这些文档里也有""它发现了我们都没意识到的盲区"
      续费逻辑一锤子买卖接班人也能用，越用越值钱
      竞争壁垒几个 prompt 能复刻数据闭环 + 追问策略沉淀
      空位定位

      这才是能打 SaaS 的产品形态。竞品（KS-Agents / Exit Insights / Morphe）大多还停在"会谈式咨询"，没有人把它做成持续运行的 Agent——真正的空位在这里。

      触发追问的 6 种信号

      信号例子
      决策点附近数据缺失"为什么 7 月 12 日突然换了供应商？前后邮件都没解释"
      多源冲突"Slack 里说要 push，但 PR 却 close 了，哪个是真实意图？"
      模式异常"你通常 review 一轮，这个 PR review 了三轮——发生了什么？"
      时间断层"这件事 6 月开始就没记录，但项目还在推进，谁在跟？"
      词汇模糊"客户『有点意见』是什么程度？要不要回退方案？"
      图谱空节点"这个人在你邮件里出现过 200 次，但没任何角色描述"

      5 种追问方式（怎么不烦人）

      方式节奏适合什么
      IM 卡片式（钉钉/飞书/企微）每天 5-10 条短决策、是非选择
      周会式 30 分钟每周一次复杂决策、需要展开
      上下文嵌入式员工写邮件时弹"上次类似情况怎么处理的"自然、低打扰
      反例诱导不定期暴露未文档化的原则
      A/B 对比不定期挖出"什么时候用 A、什么时候用 B"

      反例诱导最强——面对真实场景人只会说"这就该这么做"，但面对反例会自然解释"因为如果反过来会有 XX 风险"。这个解释才是真正的原则。

      沉淀格式：不是 QA，是「决策原则 + 适用条件」

      不要存成 Q: 为什么换供应商？ A: 因为对方涨价 30%。要存成：

      原则：当原供应商单方面提价 >20% 且无 SLA 改善，3 个月内启动替换流程
触发条件：涨价幅度、SLA 变化、替换成本评估
反例：如果对方因不可抗力（汇率/政策）涨价且 <30 天可恢复，则不启动
来源：2024-07-12 邮件 #324 + Active Inquiry 2026-05-15
泛化层级：通用（不仅限于这个供应商）

      继任者来查"供应商涨价怎么办"时，Agent 不是给他翻邮件，是给出原则 + 当时为什么 + 反例。

      员工凭什么配合（最被低估的难题）

      员工已经要走了，为什么花时间回答 Agent？尤其是被裁员、跳槽到竞争对手、对公司不满的情况。

      激励说明
      绑离职补偿完成度作为离职流程 checklist，与 N 个月补偿绑定（HR 主动推）
      绑前同事骚扰卖点是"以后老同事不会半夜 IM 你"——直击人性痛点
      绑职业声誉完成度 → 给原公司的离职评价 → LinkedIn 推荐
      绑接班人体验接班人付小钱，反向给原员工现金奖励
      绑学习沉淀"你的经验本来就该结构化"——卖给他自己未来用

      最强的是前两个——前者搭便车 HR 流程，后者击中真实痛点。

      反向接力机制（长期价值的护城河）

      如果产品做得好，会产生这样的链：

      flowchart LR A[原员工] --> B[Agent] B --> C[继任者] C -.付费追问.-> A A -.补充原则.-> B style A fill:#ede9fe,stroke:#4338ca,stroke-width:2px,color:#1c1c1e style B fill:#ecfdf5,stroke:#16a34a,stroke-width:2px,color:#1c1c1e style C fill:#fef3c7,stroke:#b45309,stroke-width:2px,color:#1c1c1e

      接班人遇到 Agent 答不出的问题，可以付费向原员工发起一次追问。这给了原员工：

      • 持续被需要的尊严
      • 持续收入（哪怕几年后）
      • 强动力一开始就配合 Agent——挖得越准，以后被反向追问越少
      独有优势

      把"一次性离职交接"扭成了长期关系网络——这是 Delphi 那种 ToC 创作者模式做不到的，只有 ToB 离职这个场景天然成立。

      对评测的反馈意义（连接「下注一」与「下注三」）

      Active Inquiry 还有一个副产物：它产生了天然的评测数据。

      • Agent 触发追问 = 它认为这里有不确定
      • 员工的回答 = 这次不确定被怎么解决
      • 继任者后续是否还在追问同一类问题 = 知识库是否真的有用
      战略连接

      这是离职 Agent 给「数字分身评测尺」喂数据的最直接路径——也是「下注一（离职交接）」与「下注三（评测 + 透明审计）」在产品层的天然连接点。先做下注一，评测能力作为副产物沉淀；等积累足够，下注三自然成立。

      6.1 三条最实在的下注（中国语境）

      下注一 · 现金流

      国内 ToB 离职交接 Agent

      • 海外 5+ 家小厂证明商业模式跑通
      • 国内完全没有专精玩家
      • 客单价：每位离职员工 ¥5k-20k，按人头计费
      • 6 个月可拿 5-10 个付费客户
      下注二 · 高客单

      资深专家方法论提取

      • 海外只有制造业，咨询 / 医生 / 律师 / 高级工程师 / 金融分析师全空白
      • 精品咨询模式：单项目 ¥50k-500k
      • 不需要规模，靠 reference 累
      • 从一个特定行业切入（如券商研究员）
      下注三 · 长期

      数字分身评测 + 认知透明审计

      • 当前所有玩家没人做"挖得对不对"的评测
      • 是「认知透明」概念的具体落地
      • 短期赚钱难，长期是基础设施
      • 两条腿走：做能力 → 给"下注一/二"提供能力 → 收编客户

      七技术难点与风险#

      7.1 技术难点

      难点现状影响
      隐性知识抽取显性知识能挖、隐性知识需要主动追问 + 反例诱导决定"决策能不能复现"
      数据连接器每接一个新数据源都是脏活决定能不能规模化
      评估金标学术有方法、行业没标准决定客户敢不敢用
      风格漂移时间长了会失真决定 Agent 能不能长期用
      多源冲突邮件说 A、聊天说 B，怎么裁决决定可信度
      去 PII法规越来越紧决定能不能进合规客户

      7.2 商业 / 法律风险

      风险程度缓解
      个人信息保护法 / GDPR高私有部署 + 数据出境控制 + 知情同意
      数据归属：员工还是企业中合同条款 + 离职时双向签字
      死者无法同意（临终类）不做就没有直接避开这条赛道
      被大厂收编中走 ToB 垂直 + 高客单逃离
      激活空间技术成熟后被大模型厂吃掉中把竞争力放在应用层和评测层而不是底层算法

      7.3 观察信号

      以下事件发生意味着需要重新判断：

      • Delphi.ai 或 Personal.ai 推出企业版离职交接专用产品
      • 国内出现第一家专精"AI 离职交接"的公司，融资到 A 轮
      • 大模型厂商（OpenAI / Anthropic / 字节）推出"基于历史对话训人格"的官方 SDK
      • Mem0 / Letta 推出企业知识沉淀产品线
      • 中国劳动法 / 个人信息保护法对"员工数字遗产归属"出具体解释
      • 出现专门做 Person Mining 评测的公司
      • 出现 Person Mining 数据连接器开源标准

      八开放问题#

      1. 隐性知识真的能被挖出来吗？还是只能挖到"被表达过的"知识？ —— 这是技术上的根本不确定性。如果不能，Person Mining 的天花板就是"显性知识 + 风格"。
      2. 评测金标怎么定？ —— 没有金标，企业不敢付钱。需要和早期客户共建"什么叫挖准了"。
      3. 数据归属：员工写的邮件、PR、文档，是员工的还是企业的？ —— 法规模糊，合同先行。
      4. 激活空间技术成熟后，会不会把所有产品层 collapse 成"上传数据 → 拿到人格向量"？ —— 如果会，应用层和评测层是仅存的护城河。
      5. 国内监管路径：员工离职后企业用 AI 保留其工作模式，算"个人信息处理"还是"商业秘密保护"？
