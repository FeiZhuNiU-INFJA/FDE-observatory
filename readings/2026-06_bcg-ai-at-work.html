      信源：BCG · AI at Work, June 2026 (4th edition) · PDF

      核心结论：BCG 第四版（n=11,749）的核心信息不是「AI 又进步了」，而是 adoption 这道题已被做完——74% frontline 已是常规用户（+23pp）。真正的瓶颈彻底从技术/工具转移到组织与管理：省下的时间不会自动变成价值（42% 每周省一个工作日，但 66% 没人告诉他们时间该干嘛），只有 end-to-end 流程重设计 + 战略清晰度（CEO 亲自 own） 才能把省时变成 measurable impact。最反直觉的一拳：战略清晰但工具有限的企业（80% measurable impact）反超工具很强但没方向的企业（60%）；蜜月期靠新鲜感，成熟期靠「言行一致」。Agent 已落地（接入翻倍、61% 信三年干半活）但治理真空（50%）。对 FDE：卖价值不卖 adoption、交付侧主推流程重设计、治理 + eval 是新咨询面、培训是最便宜的入口。
      一页要点
      • 瓶颈转移：adoption 已解决（74% frontline 常规用户），难题现在是组织与管理，不是技术
      • 省时 ≠ 创值：42% 每周省一个工作日，但 66% 无指引、58% 未转投战略工作——时间「漏出」组织
      • 重设计溢价：Reshape/Invent 企业在省时 / 愉悦 / impact 上全面领先 Deploy-only 约 +18~24pp
      • 策略胜工具：战略清晰 + 低工具（80%）> 低清晰 + 强工具（60%）；clarity +25pp vs tools 仅 +5pp
      • 蜜月期会过：早期「认知负荷」是愉悦驱动（+43pp），成熟后被「战略清晰度」取代（+24→+30pp）
      • 言行一致 + 追踪价值：分别是最强愉悦杠杆（+28pp）与最强影响杠杆（+22pp）
      • Agent 落地但治理滞后：接入翻倍（13→30%），但 50% 无「人×AI」治理、47% 担忧问责

      二瓶颈转移：从技术回到组织#

      From Tech to Org · 工作本身在被改写

      当 74% frontline 已是常规用户，技术不再是门槛。真正在变的是工作本身的性质——技能预期、任务边界、人与 AI 的分工。报告列出 AI 已经改变的六个工作面向（同意占比）：

      AI 改变了什么同意不同意
      岗位的技能预期已变72%13%
      接管了简单任务，留下复杂高风险工作67%16%
      「够好」的门槛被抬高60%17%
      花更多时间审查 / 纠正 AI 输出52%22%
      角色转向「管理与指挥 AI」47%23%
      决策权增加41%24%

      关键信号：72% 的人说岗位技能预期变了，近半数说角色已转向「管理 / 指挥 AI」而非亲自干活。这不是工具升级，是工种重写——而工种重写是 CEO 和 HR 的活，不是 IT 部门的活。

      The Leak · 省下的时间会溜走

      省下的时间会「漏出」组织——除非被主动追踪与再投资。报告的处方直白：衡量价值，而非 adoption。

      层级差也很刺眼：管理者与领导者感知到的指引更清晰（比 frontline 高 +14pp / +22pp），impact 自评也更高。也就是说，离权力越近的人越觉得「已经在用得好」，而真正干活的 frontline 既缺指引、也没把时间用到刀刃上——价值捕获的瓶颈在最底层，但决策权在最顶层。

      四重设计溢价：Reshape / Invent 全面领先#

      Deploy → Reshape → Invent · 越往上越值

      读法 caveat

      「Reshape/Invent 更成功」可能是反向因果：本来就战略清晰、敢投人、有 roadmap 的公司才去做 Reshape/Invent，而非做了才变好。报告用同一组驱动因子（清晰 roadmap + 投人）解释成功——两边其实是同一拨公司，这是相关性，报告按因果讲。

      五策略胜过工具：一张 2×2 说清全部#

      六愉悦悖论与蜜月期#

      The Joy Paradox · 又好又累，且会过期

      价值与愉悦不是 tradeoff——它们被同一组力驱动。但 AI 让工作「又好又累」：三分之二常规用户更享受工作，同时四成多认知负荷上升（领导者尤甚，48%）。

      67% 常规用户报告
      工作满意度上升（领导者 77%）
      41% 同时报告
      认知负荷上升（领导者 48%）

      更关键的是愉悦的来源会随时间翻转。用 AI 不到半年的新人，靠新鲜感和脑力拉伸获得愉悦；用满一年后，「认知负荷」从驱动变成负担，唯一能持续撑住愉悦的是战略清晰度。

      八报告落点：CEO 五条命令#

      Five Imperatives · 报告给最高管理者的处方

      1. 换计分板

        衡量价值而非 adoption。省下的时间会漏出组织，除非被追踪与再投资。

      2. 战略清晰度置顶，且亲自 own

        这不是传播任务，是领导姿态。亲自 own 的 CEO 在价值 / 愉悦 / 信任上全维度领先。

      3. 投 end-to-end 重设计，而非更多工具

        大多数公司仍把 AI 当个人生产力工具；更重要的变化是集体的——任务如何跨组织流动。

      4. 把人放在重设计的中心

        让人参与 shaping change，而非被 present。留住人的是"AI 如何帮我成长"，不是"我变快了多少"。

      5. 当成移动靶治理，不是一次性项目

        技术变得比任何公司都快。设一个轻量、常设的治理，反复校验什么有效、重测价值、随模型演进调整。

      九FDE / ToB 映射#

      Field Mapping · 落到客户现场

      1. i卖"夜间照明"而非 adoption：客户买的不是使用率、不是省时，是 measurable business outcome。报价 / SLA 挂"价值被追踪 + 再投资"，而不是 token 配额——和公用事业论完全同向。
      2. ii交付侧主推 end-to-end 重设计：纯"部署工具"型 POC 正被客户内部判为低价值（p9 那组 +18~24pp 差距就是话术弹药）。FDE 方案要主动包流程重设计，而不是只塞模型。
      3. iii治理 + eval 是新咨询面：50% 治理真空 + 47% 问责担忧 = "人×AI 治理 + 可问责 eval + 多供应商可迁移"是 ToB 现在最缺、也最愿付费的缺口（正好对上 p22"战略清晰度"）。
      4. iv培训是最便宜的入口：88% 需求 vs 36% 满足，这 52pp 缺口是进客户成本最低的楔子——先卖再培训，再上重设计。
      5. v帮客户把 CEO 拉进驾驶位：报告反复证明"CEO 亲自 own"是全维度领先变量。FDE 推进卡在 IT 时，数据弹药是"这是 CEO 议程，不是 CIO 议程"。
      6. viAgent POC 必须自带治理 SKU：技术翻倍落地但治理真空——交付 agent 时把"人机协作规则 + 问责 + 审计"打包，否则就是给客户埋雷。

      十待验证与批判#

      Open Threads · 留给后续

      1. self-report 偏差有多大？"省一个工作日 / measurable impact"若用客观产出校验，会缩水多少？
      2. Reshape/Invent 的溢价里，因果与选型效应各占几成？需要纵向追踪同一公司转型前后的面板数据。
      3. "61% 信 agent 三年干半活"是预期还是能力？把它当情绪指标而非预测，那真实自动化潜力曲线长什么样？
      4. "战略清晰度 +25pp"在不同行业 / 公司规模 / 监管强度下是否稳健？金融与公共部门的"清晰度"是否更难落地？
      5. 蜜月期翻转是否可被组织主动延长？"言行一致 +28pp"能否补偿认知负荷的退场效应？
