      论文：arXiv:2603.28990 · Drop the Hierarchy and Roles（2026-03-30，投 IEEE Access，6 图 9 表） · 全文 HTML

      核心结论：多 Agent 系统不该照搬人类组织的「固定角色 + 层级」。一项 25,000 任务的实验揭示了内生性悖论（Endogeneity Paradox）——既不是最强外部控制（Coordinator 中央协调）最优，也不是最彻底的自主（Shared 全自治）最优。胜出的是混合的「序贯 / Sequential」协议：固定发言顺序，但角色由 Agent 自主选择；它比中央协调高 +14%、比全自治高 44%（Cohen's d=1.86）。更反直觉的是：自组织是强模型的特权——弱模型放开自主反而更差。给 Agent 三样东西——使命、协议、足够强的模型，而不是一个预设角色。
      一页要点
      • 内生性悖论：质量随自主度呈倒 U 型，峰值落在「最小脚手架 + 最大角色自主」的 Sequential，而非两极
      • 信息论根因：Sequential 让每个 Agent 看到前序同伴的已完成产出（事实），优于意图（可能变）、历史（可能过时）、他人计划（受限于一人）
      • 自组织是强模型特权：Claude 放开 +3.5%，GLM-5 放开 −9.6%——低于能力阈值时刚性结构反而更好
      • 涌现：8 个 Agent 自发产生 5,006 个角色（RSI→0）、主动弃权（38/60 自愿）、浅层级（深度仅 1→2）
      • 规模亚线性：4→256 Agent 质量不降（p=0.61），成本仅 +11.8%；N=256 时约 45% Agent 自发闲置
      • 实操配方：使命 + 协议 + 强模型；并组合模型（开源 DeepSeek = 95% Claude 质量、24× 便宜）

      一背景：横纵两条路，与一个被忽视的问题#

      The Gap · 被忽视的核心问题

      多 Agent 研究分两条路。纵向（vertical）让单个 Agent 变强——如 Meta 的 DGM-Hyperagents，靠元认知自我改造实现开放式自改进；横向（horizontal）解决一群 Agent 如何协作，主流框架（ChatDev、MetaGPT、AutoGen、AgentVerse）几乎都照搬人类组织：固定角色、中央任务分配、刚性层级。

      但论文指出一个根本错位：LLM Agent 与人类员工本质不同——角色切换零成本（瞬时从架构师变分析师）、可瞬时处理全部组织上下文、空闲时边际成本为零。把人类组织的角色与层级照搬到这样的实体上，等于把人类本没有的局限强加给它们。

      被忽视的问题

      什么样的协调架构能在质量 / 成本 / 可扩展性 / 韧性之间取得最优权衡？此前无人系统性地沿「外部强加 → 内生涌现」的谱系去比较协议、把规模推到 256 Agent、横跨 8 个模型并实现零样本运行时自组织。本文补上这块。

      二内生性悖论：最大控制与最大自由都不赢#

      The Endogeneity Paradox · 本文中枢

      论文区分两种协调：外生（exogenous）——结构由外部强加（如 Coordinator 由 Agent 0 给所有人派角色）；内生（endogenous）——结构由系统内部涌现。中枢发现是内生性悖论：质量与自主度之间是非单调的倒 U 型关系——既不是最强控制，也不是最彻底自主，而是两者之间的混合最优。

      ← 外生 exogenous · 协调控制 ▲ 峰值：最小结构 + 最大角色自由 内生 endogenous · 自主自治 →

      同模型、同任务下，协调机制对质量的影响堪比甚至超过模型选择。倒 U 的峰值不在两极，而在「固定顺序 + 自主选角」的 Sequential。

      2.1 为什么是 Sequential？一个信息论解释

      Sequential 让每个 Agent 收到的是事实性的、任务相关的、累积的信息——前序同伴在本任务里已完成的产出。论文论证这种信息类型压倒性地优于其它三者：

      协议Agent 看到的信息缺陷
      Broadcast 广播他人意图（两轮：先广播意图再定）意图可能改变
      Shared 共享历史角色记录（共享组织记忆）历史可能过时、不适用当前任务
      Coordinator 中央单一协调者的计划受限于一个 Agent 的判断
      Sequential 序贯前序同伴的已完成产出（O(N) 延迟，见局限）
      类比体育选秀：每支球队在 knowing all previous picks 的情况下选择，自然补齐互补位置——无需中央规划。悖论在于：最小结构释放最大涌现——一个简单约束（固定顺序）解锁了任何显式设计都达不到的角色分化、主动弃权与使命对齐。

      3.1 Sequential 如何运转：按序补位，末端可弃权

      Inside Sequential · 胜出协议的内部机制

      把胜出的 Sequential 拆开，它就是一条规则 + 三个自主决策。规则只有一条：固定顺序 A₁→A₂→…→Aₙ（整套机制里唯一的外生结构）。轮到 Aᵢ 时，他读到的是前面所有人在本任务里已完成的产出——事实，而非意图、历史或某人的计划。在这份上下文上，Aᵢ 完全自主地做三件事：选角（看缺什么补什么）、参与或弃权（自评边际价值够不够）、贡献（产出立刻对后续所有人可见）。没有主管、没有预设角色表。

      那顺序按什么排?

      顺序是外生给定的——agent 只决定「演什么角色」，不决定「什么时候上」。论文把 ordering 当成固定实验常量（像 temperature 一样给定、不优化），既未披露确切排序规则，也未做顺序消融（倒序 / 每任务重排，质量会变吗？）。合理推断：实验里 N 个 agent 是同一模型的 N 份拷贝、无能力梯度（角色当场涌现，RSI→0），所以「谁排第一」没有能力依据可排——顺序实为任意 index；正因 agent 可交换，论文把它当不重要的常量略过。唯一概念上不对称的是信息量（A₁ 只看到任务、Aₙ 看到全部），但期望意义上可交换，故非论文的杠杆。

      关键 nuance · 合作 ≠ 对话

      Sequential 下的「配合」是通过共享产物的间接协调（stigmergy 式）：Agent 之间不互相说话，而是读「目前共同作品的状态」、写下一段增量。这带来一个结构性后果——信息不对称：A₁ 信息最少（只有任务），Aₙ 信息最多（看过所有人），于是自然形成「前期打地基、后期补空位 / 修剪」的分工。也正因协调靠本任务内的累积产出，它不依赖跨任务记忆（不像 Shared 依赖会过时的角色历史）——这是它信息更新鲜、质量更高的根因之一。

      四自组织是强模型的特权#

      The Capability Threshold · 自主的分水岭

      论文最实操、也最反直觉的发现：自组织不是免费的——它要求模型足够强。在自由（自组织）与固定角色两种运行方式间比较，出现了反转效应：

      强模型 · 自主有帮助
      Claude Sonnet 4.6
      +3.5%

      自由形态 Q=0.594 > 固定角色 0.574——能力足以支撑自我反思与协调。

      弱模型 · 自主有害
      GLM-5
      −9.6%

      自由形态 Q=0.519 < 固定角色 0.574——低于阈值，刚性结构反而更好。

      阈值要求三种能力同时具备：

      1. 自我反思——能评估自身胜任度（Claude 主动弃权率 8.6%，GLM-5 仅 0.8%）；
      2. 深度推理——多步逻辑链；
      3. 指令遵循——精确遵守协调协议。
      部署门禁

      不要把自组织无差别套到任何模型上。低于能力阈值时，放开自主会让质量反向劣化——此时保留刚性结构 / 人工 gate 才是对的。好消息：阈值会随基座模型变强而下移，自主协调的适用范围将持续扩大。

      Emergence · 无设计者而自发生成

      给最小脚手架后，系统自发产生三种组织现象——没有任何外部设计或预设角色：

      动态角色分化

      主动自弃权

      Sequential 中 38/60 非贡献者是自愿退出（内生）；Coordinator 中 60/60 全被指令排除。自愿弃权对应更高质量（0.875 vs 0.767）。

      浅自组织层级

      从 4 到 64 Agent，层级深度仅从 1.0 涨到 2.0——系统偏好扁平结构，至多两层管理，无需外部指令。

      六三环宪政框架：给自组织画边界#

      Constitutional Framework · 有界自主

      自组织需要边界：近乎全自主的 Shared 最差（Q=0.503），有界自主的 Sequential 最好（Q=0.875）。论文据此提出三环宪政框架——越靠近「为什么」越多人类掌控，越靠近「怎么做」越多系统自主。

      Ring 1 不可变核心 · 仅人类：使命、价值观、弃权权。错误会在全系统级联。👤 human only
      Ring 2 标准 · 人 + 系统：指标、治理、审计。系统提议，人类批准。👤⚙ human + system
      Ring 3 协议 · 全自主：协调参数、批次大小、阈值。完全自主 + A/B 测试自调优。⚙ autonomous
      原则：越靠近「为什么」，越多人类掌控；越靠近「怎么做」，越多系统自主。

      七实操配方：使命、协议、模型——不是角色#

      The Recipe · 落地四条

      1. 1定义使命与价值观，而非角色分配。LLM Agent 不受身体或固定技能束缚——预设角色是把人类局限复制到没有它的实体上。给使命 + 自由，使命相关性达 4.00/4.00；给角色描述反而达不到。
      2. 2选对协议。强模型中协议选择解释 44% 质量变异（Sequential vs Shared，同模型同任务）——协议是集体能力的放大器。
      3. 3投资模型质量，而非 Agent 数量。64→256 Agent 无质量提升（p=0.61）却 4.6× 成本；模型间质量极差 174%——「乐手」比「座位数」重要。
      4. 4组合模型，别只选一个。DeepSeek 达 Claude 95% 质量、24× 便宜；强模型打 L3–L4，高效模型打 L1–L2，没有任何单一模型在所有维度称王。
      判断

      「放弃层级与角色」不是无政府，而是有界自主。最小结构（固定顺序）+ 最大角色自由 + 足够强的模型，三者乘性叠加，胜过任何精心设计的组织图。编排的本质，正从「指挥」转向「设定让涌现发生的条件」。

      八FDE / ToB 映射#

      Field Mapping · 落到客户现场

      1. i角色预设是反模式：客户多 Agent POC 里常见的「planner / coder / reviewer」固定角色设计，应让位于使命驱动 + 协议选择——这直接挑战现有框架（MetaGPT / ChatDev 类）的默认假设。
      2. ii协议优先于拓扑：选型应问「协议是否可审计、弃权是否可追溯、产出是否可回滚」，而非一味堆 Agent 数或画更复杂的流程图。
      3. iii能力阈值 = 部署门禁：弱模型场景必须保留刚性结构 / 人工 gate，否则自主反而劣化——这条直接影响交付 SLA 与质量承诺。
      4. iv使命契约 = 新的需求规格：把「mission + values + 弃权权」写成不可变核心（Ring 1），对应企业的治理、合规与红线边界。
      5. v多模型成本策略：强模型攻坚 + 开源模型摊量，是 ToB 降本的可行路径（24× 成本差，95% 质量）——把模型组合写进报价与架构。

      九局限与保留#

      读这份结论前先看这五条
      • LLM-as-judge：所有质量评估依赖 LLM 评判（GPT-4o / GPT-5.4），无人审，可能偏好冗长回答；计划做 50–100 任务人工校验。
      • 评判模型跨系列变更（GPT-4o → GPT-5.4）：系列内比较有效，跨系列绝对值不可直接比。
      • 合成任务：L1–L4 均为合成，缺真实业务工作流验证。
      • Sequential 延迟 O(N)：大规模可能受限；分批 Sequential（K 个并行→下一组观察全部前序）是缓解方向。
      • API 稳定性 / 限流：部分模型因限流降低完成率，引入潜在选择偏差。（多重比较经 Bonferroni 校正后主结论仍显著：α=0.0025。）
