      协同设计的 100×：为什么算力叙事都在看错方向

      Dylan Patel（SemiAnalysis 创始人）× Sequoia · 70 分钟深谈 · FDE 结构化拆解

      2026-07-06 跨市场 硬件 / 推理 / 供应链 / 多极 可对外 时机：进行中 信心：高

      Why Hardware-Software Co-Design Is AI's Real 100x: Dylan Patel of SemiAnalysis

      Sequoia Capital · Shaun Maguire & Sonya Huang 主持 · 时长 1:10:15 · 原视频 ↗

      一句话结论：过去三年 AI 效率的真实增益不来自 GPU 换代——它来自把「硅片形状 × 内核形状 × 模型形状」三层一起优化。单独看每一层，都只有 2× 左右的进步；耦合起来，是 相同质量下 60× / 年的 token 成本下降。看不懂这一点，就会用「一块芯片 vs 一块芯片」的错误单位去分析一个已经进入三体协同的行业。

      一为什么现在读这段视频#

      Framing · 定位

      Dylan Patel 不是普通评论员。他把 SemiAnalysis 做成了年营收破亿美元、90 人团队的独立研究机构：一半是横跨供应链的工程师，一半是对冲基金背景的分析员。这是行业里罕见的「同时懂化学、光刻机、内核和 P&L」的建制。这场对谈的价值不在他讲了什么新八卦，而在他把过去两年算力叙事的失焦点系统化地纠正了一遍。

      如果你在过去 12 个月听到过下面这些论断——

      • 「CUDA 是 NVIDIA 最深的护城河」
      • 「AI 没有 ROI，会崩」
      • 「大家会转向 TPU / Trainium / Cerebras」
      • 「模型进步已经放缓」
      • 「hyperscaler 会统吃云生意」

      ——Dylan 的每一条回应都值得写进这份笔记。他不否认这些命题的表面观察，但他给了一个更基础的解释框架：协同设计（co-design）。

      “The real breakthrough is when you leapfrog a few layers, you co-optimize and co-design them, and now all of a sudden you've taken what could have been a 2× here, 2× here, 2× here, and instead of being multiplicative to 8×, it's actually 100× because you've optimized across all three layers.” —— Dylan Patel · 22:15

      二核心论点：三层协同放大器#

      The Real 100× · 100 倍从哪里来

      大多数分析把 AI 的效率进步拆成三个平行的赛道——硅片、内核、模型算法。每一层都能独立跑出 2 倍的年化增益。三个 2× 叠加起来，直觉上是 8 倍。这已经很不错了。

      但真实数据是 60 倍 / 年——SemiAnalysis 的活基准（InferenceX）在同等 benchmark 分数下，跑出的成本下降就是这个量级。这个数字远远超出「三个 2× 相乘」能解释的范围。

      硅片Hopper → Blackwell 内存/带宽/网络三年进步
      内核SGLang / VLM / PyTorch 每周迭代两次，speculative decoding、MTP
      模型Sparse MoE、attention 变体、expert shape 精雕

      协同层不是把三个 2× 加起来——它把每一层的 shape 都刻进另外两层。

      Dylan 拿 DeepSeek 举例，直白到几乎粗暴：DeepSeek V3 的所有专家形状都是为 Hopper 定制的，V4 为 Blackwell 和华为芯片定制。这不是「顺便适配」，是从矩阵乘法单元的物理尺寸开始，往上倒推 attention 机制、collective 通信模式、arithmetic intensity——最后才决定 expert 的宽度和深度。

      协同的真面目
      • Shapes（形状）——矩阵乘法单元尺寸决定 attention 结构决定 expert 结构
      • Network IO patterns（网络拓扑）——NVLink 只连 72 卡有 switch，Google ICI 连 8000 卡无 switch，模型必须适应拓扑
      • Arithmetic intensity（算术强度）——每次内存读取要做多少浮点，决定选 batch size 还是选 speculative decoding
      • Collectives（集合通信）——all-reduce / all-to-all 的具体调度，直接反过来决定 expert 数量

      三“CUDA 护城河” 从来不是 CUDA#

      Moat Reframing · 定义纠正

      这是全片最反直觉、也最重要的一段。

      过去三年，业界普遍相信 NVIDIA 的护城河是 CUDA API 的开发者生态锁定——「你要花几年时间才能写好 kernel，所以离不开 NVIDIA」。Sequoia 的主持人 Shaun 直接问 Dylan：现在 AI 会自己写 kernel 了，这个论点是不是要塌？

      “什么叫 CUDA moat 其实和 CUDA 无关。是因为 DeepSeek、Kimi、智谱、Alibaba、腾讯、小米——所有这些公司的模型都是为 GPU 协同设计的。你把这些模型放到 TPU 上跑，很多情况下就是不行。” —— Dylan Patel · 33:20
      ✗ 常见误解

      "CUDA API 的开发者锁定"

      观点：写 CUDA kernel 门槛高，几万开发者被锁定；模型公司离不开 NVIDIA。
      问题：全世界真正需要写 kernel 的公司只有几十家 lab；AI 又特别擅长写代码，这层锁定确实在解体。

      → reframe
      ✓ 真正的护城河

      下游开源模型的 GPU-shape 锁定

      中国实验室（DeepSeek/Kimi/Zhipu/阿里/腾讯/小米）把模型 shape 深度耦合到 GPU 的矩阵乘单元和网络拓扑上。整个开源模型生态都是 GPU-native 的。你想搬去 TPU？形状不对，性能崩。

      推论极其锋利：只要中国开源模型继续把最好的模型 co-design 在 GPU 上开源，NVIDIA 的护城河就自动持续——即使 Anthropic、OpenAI 内部改用 fork 版 PyTorch，即使 kernel 被 AI 写代码打穿，形状锁不动。Google 想反击？「唯一办法是自己开源真正好的 Gemma 模型，让下游反过来把 shape 锁到 TPU 上」——但 Gemma 显然还没做到。

      对 FDE 的解读

      当我们跟踪 frontier 板块的 NVIDIA / Google / Anthropic 信号时，「CUDA vs TPU 谁赢」是错的问题。正确的问题是：下游开源生态在往哪个 shape 收敛？——过去 18 个月的答案不含糊：GPU shape。这决定了未来 3 年推理成本曲线的落点。

      四模型分叉正在把硬件也分叉#

      Sparse ↔ Dense · 三条独立演化路径

      Dylan 拆开三家头部 lab 的模型形状——第一次让我看清了 为什么他们看起来「都在做 LLM」，但硬件绑定完全不同。

      OpenAI · 更稀疏

      非常稀疏的 MoE 结构，天然贴合 Hopper 的矩阵单元和 NVLink 拓扑（72 卡域）。想让 GPT-4o 换到 TPU？“terrible decision.”

      Anthropic · 稠密度居中

      模型形状能同时被 TPU（训练）和 Trainium/GPU（推理）消化。这解释了他们能拿 SpaceX 的 GPU 深度折扣，同时 Q2 已经净利润为正。

      Google · TPU-native

      不只是 Broadcom 一家 TPU，而是 Broadcom / MediaTek / 内部三条独立架构并行。Gemini 每代重构对齐下代 TPU——把 co-design 推到极致。

      “硬件是好硬件，就是跑不动这个模型”——这句话在几年前听起来不合逻辑，现在是产业常识。TPU 客观上是极强的芯片，但它跑 DeepSeek 就是不行；反过来，某些模型在 NVIDIA 上就是浪费。硬件的绝对性能已经不足以作为选型依据，只有「你的模型和它的形状能不能对上」才算数。

      这也回答了一个业界长期争论：「每个 lab / hyperscaler 是不是都会自研芯片？」Dylan 的答案是“每个人都会试，然后大多数人会放弃”——不是因为芯片不重要，而是你不知道一年后的模型架构长什么样。你花五年时间赌一个 ASIC 的最优形状，模型架构可能已经换了 attention 机制。所以未来的稳态是：hyperscaler 都会有自研 ASIC 承担已知负载，但依然要买大量 GPU 作为「不知道未来是什么」的通用备份。「Google 用 11 美元 / 小时向 xAI 租 GPU」就是这个逻辑的活证据。

      五一切推理决策的母图#

      The Master Curve · Throughput × Interactivity

      Dylan 花了大约五分钟解释一个他反复挂在嘴边的曲线——吞吐量 × 交互性（throughput × interactivity）。他说：「硬件、基础设施、模型、应用层的所有决策，都下游于这条曲线。」

      关键：同一块硬件、同一个模型，能停在这条曲线的任何一点。停在左上（Batch）——4× 便宜；停在右下（Fast Mode）——4× 贵。Anthropic 的 Fast Mode、OpenAI 的 Priority Queue 都是这条曲线的商业化产物；未来一定不是「一价制」，而是「曲线上选点」。产品经理如果不理解这条曲线，就无法为不同 workload 定不同的价。

      对 FDE 的解读

      我们做 Agent / 应用侧决策时，很容易默认「token 就一个价」。真相是：token 有三条以上不同的价——批处理级、常规级、极速级。做 coding agent 后台批量修 bug 应该走批处理最贵档；做 real-time 交互体验时才为 fast mode 付溢价。产品的一次 UX 决策，直接决定你在推理曲线上停哪一点，也直接决定单位成本差 4–10×。

      六InferenceX：活基准与 60× / 年#

      Living Benchmark · 死基准的告别

      SemiAnalysis 花了大力气做了一件事：把「推理性能」从时间点快照变成连续测量。

      60×
      annual · same quality
      “相同 benchmark 分数下”的 token 成本，年化下降 60 倍——这不是硬件单独跑出来的，是三层协同的复合曲线。
      Intelligence per watt 同期下降约 40×（部分是非功耗改善）。

      为什么需要「活」基准？因为软件每周更新两次：PyTorch、VLM、SGLang、驱动、编译器；模型每周有新的（Qwen、Moonshot、DeepSeek、GPT-OSS、Nemotron）。传统 benchmark 一发布就过期。

      怎么做到的？让所有主要 hyperscaler（CoreWeave、Crusoe、Nebius、Oracle、Microsoft、AWS、Google、OpenAI）捐赠合计 > 5000 万美元的硬件（等 TPU / Trainium 上线后是 1 亿+），15+ 芯片型号，跑最新模型，扫全 Pareto 曲线，把最优配置的容器开源。

      这件事的深意是：「行业公用的活坐标系」这个位置以前是空的，SemiAnalysis 站到了这个位置。任何芯片厂商想证明自己「不亚于 NVIDIA」，都得先在这个坐标系上站住。这本身是巨大的话语权杠杆。

      七算力紧张为什么会持续#

      Compute Crunch · 供需失衡的真正原因

      这是我最想让读者停下来想清楚的一段。表面上：算力短缺是因为「数据中心建太慢 + 硬件供货紧张」。Dylan 给了一个更底层的原因：

      “The compute crunch is not because deployment is slow. It's because the value of useful work AI can do is expanding faster than compute is growing.” —— Dylan Patel · 46:30（意译）

      翻译成 FDE 惯用的判断语言：模型每一代把「AI 能干的有价值工作」的 TAM 都乘以一个大数。Opus 4.5 到 Fable 5 / Mythos 5，能干的任务数量和价值不是 2 倍，是数量级级别的跃升。而全球算力半年里可能只增长 30–50%。供需缺口不是暂时的施工延误，是「模型 TAM 增长速度 > 算力增长速度」的结构性错配。

      只要模型还在进步，crunch 就会持续。Anthropic Q2 已经净利润为正（不含股权激励），Opus 48 的单 token 毛利 >80%——他们能以任何价格回收 GPU 成本，因为每一张 GPU 都能立刻转化为高毛利 token 卖出去。这个飞轮直到模型进步停滞才会停。

      下一个问题：模型进步会停吗？Dylan 的观察：不但没停，反而在加速。因为「模型帮工程师写下一代模型的基建」——这是一个不严格意义上的 pseudo-recursive self-improvement 循环，Codex 5.5 出来之后特别明显。

      八Jensen 的多极棋盘#

      Multipolar Play · 为什么 NVIDIA 疯狂投 Neocloud 和 Neolab

      Jensen 最近做了三件让财务分析师看不懂的事：向 Neocloud（CoreWeave、Crusoe、Nebius）猛砸订单和股权、投一堆看起来 ROI 不清的 AI lab、公开为中国实验室的开源模型站台。

      Dylan 一句话讲清楚：他要一个多极世界。因为在「hyperscaler 三巨头 + OpenAI/Anthropic/Google 三巨头」的两极世界里，NVIDIA 是死的。

      Neocloud——绕过 hyperscaler 的算力分发；CoreWeave 已上市，Crusoe / Nebius 快速拉规模
      Neolab——绕过 OpenAI/Anthropic 的模型话语权；Thinking Machines 半年做到几亿 ARR
      China Open——把 GPU-shape 锁进全球开源生态，是最强杠杆

      五年后，如果这三条支线都长起来了：Google TPU 弱一点、Amazon Trainium 弱一点、更多 inference 跑在非闭源模型上——NVIDIA 就赢了。**这不是慈善，是理性极致的地缘博弈**。「Jensen 爱中国实验室」的真正原因就在这里。

      对 FDE 的解读

      我们跟踪 oss 板块（开源拆解）和 frontier 板块（厂商信号）时，要把「Jensen 的钱流向哪里」当作一级信号——那是他希望多极世界长在哪的坐标。Neocloud 的下一个赢家、Neolab 的下一个 Thinking Machines、中国开源模型的下一个 DeepSeek——大概率会先出现在 Jensen 的 portfolio 里。

      九能源、太空与 10 年展望#

      Long-tail Bets · 长期赌注

      9.1 能源瓶颈没你想的那么硬

      Dylan 举了一个特别工程师的例子：美国有制造数百万台柴油卡车发动机的产能，简单改装为天然气版本 + 反接电动马达发电，就能在数据中心旁快速部署 backup 甚至主力发电。「找个汽车修理工来维护就行」。这不是理论——是有人在做的方案。能源紧张的很大一部分，是 “好的解法还没被工程化”，不是物理墙。

      9.2 硅片功率密度的物理墙正在被撬

      过去 20 年，数据中心/桌面芯片一直卡在1 瓦 / 平方毫米。Blackwell 到 Rubin Ultra 是把芯片做大到 1400W → 2000W → 4000W——不是密度提升，是面积扩张。真正的下一步：让每平方毫米消耗超过 1 瓦，代价是更复杂的热管理和电学干扰。这会带来硅片数量的净减少。

      9.3 HBM 直接堆到芯片上

      过去 5 年 HBM 进步靠更多层堆叠 + 更快接口，但根本 cell 结构还是 40 年前的 DRAM。接下来是把 HBM 直接堆到逻辑芯片上（不是旁边）——带宽爆炸。有几家 startup 在赌这条路。

      9.4 太空数据中心：2030 前 < 1%，2040 后 > 50%

      短期（3–5 年）：不重要。长期（20 年）：绝大多数增量算力会在太空——这是 Dylan 的判断。SpaceX 的星链网络经验 + 特斯拉的电池/热管理经验，让他们在这条路上有独一档的优势。

      十给 FDE 的落地判断#

      Judgment Desk · 判断台清单

      已经能行动的判断
      1. 1「一价制」将破裂——推理定价会分化成 Batch / Normal / Fast 三条曲线。做 Agent 后台任务时用批处理档，能省 4–10×；做实时体验时才为 Fast 付溢价。这是产品经理下一年必须内化的推理经济学。
      2. 2CUDA 护城河实质是「下游模型 shape」，不是 API——判断 GPU 生态强弱，看开源模型（DeepSeek/Qwen/Kimi）的 shape 在往哪个硬件收敛。目前答案不含糊：GPU。
      3. 3算力紧张会持续到「模型 TAM 增速 < 算力增速」为止——这个交叉点不会在 2027 年之前出现。规划算力预算按「持续短缺」做，别按「一年缓解」做。
      4. 4Jensen 的 portfolio = 多极信号灯——Neocloud 的下一个赢家、Neolab 的下一个 Thinking Machines、中国开源的下一个 DeepSeek，会先出现在 NVIDIA 的投资列表里。这是最便宜的一级信号。
      5. 5「哪家 lab 用哪种硬件」不再是选型问题，而是模型架构的映射——OpenAI 的稀疏 → GPU；Anthropic 的中密度 → 三栖；Google 的 TPU-native → TPU。评估任何模型的成本曲线，先看它 co-design 在哪里。
      仍需观察
      1. Cerebras / Groq 在「10T 参数 + 百万上下文」时代能否守住份额——SRAM 天花板是硬约束。
      2. Anthropic Q3 是否含股权激励也转正——如果是，会成为整个训练/推理经济学的分水岭事件。
      3. Google 什么时候开源真正好的 Gemma——那会是「GPU-shape 锁」被撬动的信号。
      4. HBM 直接堆芯片的技术路线由谁跑出——带宽层的下一波 100× 就藏在这里。
