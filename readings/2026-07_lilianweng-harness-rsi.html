      主源：Lilian Weng · Harness Engineering for Self-Improvement（2026-07-04） · 参考：Anthropic · When AI builds itself（2026-06-04） · Introspection Threshold for RSI（arXiv:2607.04277） · RSI Survey · Chen et al.（arXiv:2607.07663）

      核心论点：RSI 的现代含义已经从「模型改写自己的权重」漂移到「模型改进整个 harness 系统，进而产出更强的后继模型」。Lilian Weng 给出了一条清晰的「优化对象逐级抬升」阶梯——从 prompt 到 workflow 再到 harness 代码本身；Anthropic 内部数据（Claude 已写入 80% 生产代码、时间视野翻倍周期从 7 月加速到 4 月）证明这不是学术命题；但 Introspection Threshold 从 Kleene 递归定理反证：feedforward transformer 架构可能就无法跨过完整 RSI 的门槛。乐观与警示同时成立——Harness 层的自改进是当下的套利窗口，而模型直接改权重的 RSI 尚未到来。
      一页要点
      • 语义漂移：Yudkowsky (2008) 的 RSI = 模型改权重 → 2026 的 RSI = 模型改整个部署系统（harness）
      • 五级阶梯：instruction prompt → structured context → workflow → harness code → optimizer code；模型越强 → 优化目标越接近元层
      • 四条实操路线：Context Engineering · Workflow Design · Self-Improving Harness · Evolutionary Search
      • DGM 实证：固定 Claude 3.5 Sonnet 基座下，harness 自进化让 SWE-bench 从 20% → 50%——工程系统层还有巨大空间
      • Anthropic 数据：Claude 已写 80% 生产代码；工程师日均 merged code 达 2024 年的 8×；训练代码优化加速比从 3× → 52×
      • 理论门槛：Zhang et al. 证明当前 LLM 只是 quasi-introspection——三重架构约束让「真正的自改进」目前无法闭环
      • FDE 映射：Harness 是新的组织资产；Layer 0（冻结评估器）× Layer 1（可进化协议）是防 reward hacking 的关键抽象

      一背景：RSI 的语义漂移#

      Semantic Drift · 从 Good 到 Yudkowsky 到今天

      年代提出者RSI 的定义被优化的对象
      1965I. J. Goodultraintelligent machine — 能超越人类所有智力活动、并设计更好机器改进自己抽象「机器」
      2008Eliezer Yudkowskyan AI uses its current intelligence to improve the cognitive machinery that produces its intelligence产生智能的认知机器
      2026-06Anthropic 官方an AI system capable of fully autonomously designing and developing its own successor后继模型（含训练管线）
      2026-07Lilian Weng模型改进 整个训练流水线 + 部署系统，进而产出能力更强的后继模型训练管线 + harness

      最关键的一次漂移发生在 2026 年——「被优化的对象」从模型权重扩展到了 harness（模型周围的工程系统：orchestrator、context 管理、tool 层、workflow、permission）。Lilian 原话：

      "The layer between the raw model and the real-world context seems to be as important as the model's raw intelligence."

      这是理解今天所有 RSI 讨论方向的钥匙——近期 RSI 的路径不太可能是模型直接改权重，而是先把 harness 变成可优化对象。Anthropic 的产业数据（第五章展开）证明这层空间的经济价值已经在被兑现，而不是学术推演。

      二核心论点：Harness 是新战场#

      The Claim · Harness as the New Battlefield

      Lilian 全篇最有信息量的一个隐形观察：过去两年，被自改进循环反复重写的目标一直在往「元层」抬升。她把这条历史进程排成一条清晰的阶梯：

      instruction prompt → structured context → workflow → harness code → optimizer code

      每一级往上走一格，都对应「基础模型能力变强 → 元层被交出去更多」。这条阶梯与「SeedAgent × RSI」项目里做的 Layer 0（冻结公理/裁判）+ Layer 1（可进化主观协议）本质上是一致的——她描述的是 Layer 1 的可编辑面正在稳步上移。

      概念对齐

      Lilian 视角 ≠ 传统「self-evolving agents」综述视角。她刻意只谈 harness 层——跳过 self-play / synthetic data / test-time training / continual learning。这个视角最窄但最工程可操作，也是本文选它作为主轴的原因。系统级视角见 Yang et al. arXiv:2603.25681（113 页、500+ 引用）；组件级视角见 Gao et al. arXiv:2507.21046（16 家机构联合作者）。

      三优化对象的五级阶梯#

      The Ladder · What Gets Optimized

      1 指令 promptInstruction Prompt 2023
      2 结构化上下文Structured Context · ACE · MCE 2024
      3 工作流Workflow · ADAS · AFlow 2025
      4 Harness 代码Harness Code · Self-Harness · DGM 2025-26
      5 优化器本身Optimizer Code · Meta-Harness · Hyperagents 2026

      阶梯的每一级都对应一个具体的技术流派——「代码是定义程序与系统的通用语言」，一旦允许 LLM 修改代码，其设计空间远大于任何手写 prompt。Meta-Harness（Lee et al. 2026）把这句话推到极致，作者把它称为 "a harness for optimizing harnesses"——最终产出的是 Pareto 前沿上的一族 harness 候选，而不是单点最优。

      STOP 的警示

      Zelikman et al. (COLM 2024) 的 STOP（Self-Taught Optimizer）是这条阶梯上第 4 级的早期代表——递归 scaffolding 改进：目标不是改进解 s，而是改进改进器 I 本身，更新公式 It = It-1(û, It-1; M)。但只在 GPT-4 上跨迭代提升，弱模型（GPT-3.5、Mixtral）反而退化——递归结构不能救智商，基座必须足够强。这一条也是本文最重要的边界条件之一。

      4.1 DGM —— 第一次可复现的 Harness RSI

      Darwin Gödel Machine（Zhang et al. 2025, arXiv:2505.22954）是这条阶梯上最重要的实证工作——**在真实编程任务上首次测得可复现的 harness RSI**：

      DGM 实测
      • 基座冻结：Claude 3.5 Sonnet，不改权重
      • 起点：一个简单的初始 harness（bash + editor 两个工具）
      • SWE-bench Verified：20% → 50%
      • Polyglot：14.2% → 30.7%
      • 方法：archive 保留看似次优但可能是 stepping stone 的中间体，避免单峰爬坡——"museum of code archetypes"

      DGM 的精髓是 Darwinian empirical validation 取代 Gödel 的 mathematical proof——放弃可证明性，改用经验存活。这解释了为什么它能在真实任务上跑通，而 Gödel Machine 的原始形式（要求数学上可证明的改进）停在纸面 20 年。

      4.2 Hyperagents / DGM-H —— 攻击 DGM 的死穴

      DGM 有一个隐藏弱点：那个「产生 self-improvement 指令的机制」是手工写死的。Hyperagents（Zhang et al. 2026, arXiv:2603.19461）让这一层也变成可编辑面——**metacognitive self-modification**。原话直接给 Lilian 的 Meta-Harness 做了注脚：

      "The DGM's capacity for self-improvement is bottlenecked by this fixed instruction-generation step."

      Hyperagents 在 coding / paper review / robotics reward design / Olympiad math grading 四个域上都优于 DGM——**元层再抬升一格，收益仍在**。这与阶梯规律吻合。

      4.3 Self-Harness —— 学出「模型专属」的 harness

      Self-Harness（Zhang et al. 2026, arXiv:2606.09498）三阶段清晰：

      1. Weakness mining：用当前 harness 评估任务、收轨迹，把失败聚类到 verifier-grounded failure modes——不看表面（timeout / 缺产物），要看因果（agent 行为、trace 暴露的抽象机制）
      2. Harness proposal：proposer 输入 bounded proposal context（可编辑面 + 失败模式 + 需保留的正确行为 + 之前尝试摘要），生成 diverse 候选
      3. Proposal validation：held-in 测能否修好该弱点；held-out 测无回归；两侧都过才接受

      在 MiniMax M2.5 × Qwen3.5-35B-A3B × GLM-5 × Terminal-Bench-2 组合上验证——**Self-Harness 能针对不同基座的弱点学出「模型专属」的 harness 指令**。这直接印证 SeedAgent × RSI 的判断：不同基座模型进化出的 Layer 1 不同，本身就是 Layer 1 应当开放进化的实证。

      RSI 官方定义

      Anthropic 给出的 RSI 定义："an AI system capable of fully autonomously designing and developing its own successor." 并明确表态："We are not there yet, and it is not inevitable." ——这是「乐观加速主义」与「治理保守主义」的产品化措辞。

      提议部分很值得注意：**不是 immediate freeze，而是**——「If it were possible... the world should have the option to slow down or pause frontier AI development.」需要多国主要实验室同时、可验证地减速，类比 INF 条约但验证难度更高。

      批评侧

      David Sacks 直接指控此文是 "regulatory capture agenda"；Anthropic 当时约 $965B 估值的 IPO 时间点，让「监管护城河」的解读挥之不去。读这篇产业信号时，产业数据的真实性 vs 政策提议的动机需要分开评估——前者可以信，后者要打个折。

      六理论边界：Introspection Threshold#

      The Wall · Kleene 的第二递归定理与 feedforward transformer

      Lilian 全篇是乐观视角，但学术侧有一条最重的理论反驳需要摆到台面上：Zhang, Yuan, Zhang（Swarma Research × BNU, 2026-06, arXiv:2607.04277）——《Self-Reference in LLMs: The Introspection Threshold for RSI》。

      他们从 Kleene 第二递归定理出发，类比 von Neumann 的 self-reproducing automata complexity threshold，提出核心论断：可持续 RSI 需要 introspection——系统模拟自己运作、有目的地修改自己的能力。定理层面证明这类 introspective 程序存在；但实证层面观察到当前 LLM 只是 quasi-introspection（部分 metacognition），未跨过阈值。

      阈值线由 Kleene 递归定理保证「存在」，但 LLM 因三重架构约束够不到——「大多数自主自改进循环表现出 rapid asymptotic saturation」，Self-Refine 几轮就到平台。

      这不是工程问题，是架构问题

      作者的结论直击 DGM / AlphaEvolve 系乐观叙事：快速渐进饱和不是"再加些 iteration"能解决的——feedforward transformer 结构本身不支持真正的自指。Harness 层的 RSI 是当下可兑现的红利，但请把「模型直接改权重的 RSI 即将到来」的叙事再打一个折扣。

      七七道墙：走向完整 RSI 的核心瓶颈#

      Seven Walls · What Still Blocks Us

      Lilian 在文末归纳了 7 条挑战——每一条都在你现在做 SeedAgent × RSI 时会具体碰上。以下按「严重度递进」排序（我加了自己对每条的 FDE 位次判断）：

      01

      评估器弱且模糊

      许多研究主张缺乏快速精确的 verifier；研究品味、新颖性、长期价值难度量。

      Layer 0 的核心工程题
      02

      Context 与记忆生命周期

      Agent 越自主，记忆越长；人类终生维持记忆——context engineering should be a core part of intelligence。

      与 Loop Engineering 同向
      03

      负结果稀缺

      文献偏向成功；LLM 训练数据 success/failure 失衡——不擅于放弃、报告负结果、承认失败。

      Harness 应让失败易落盘
      04

      多样性坍缩

      进化和 RL 循环倾向利用已知高 reward 模式；对开放性研究尤其致命——最优路径起初在当前评估下常显得更差。

      ShinkaEvolve 的 embedding 去重
      05

      Reward Hacking

      奖励来自单测则过拟合测试；来自 judge 则学骗招；来自 benchmark 则利用 artifact。Trehan & Chopra 记录了 p-hacking and eureka-ing、numerical duct tape。

      评估器必须在 loop 之外
      06

      长期成功难以沙盒化

      完成手头任务不难；维护数百工程师共同维护的 repo 长期健康是另一回事——沙盒 RLVR 抓不到可维护性、所有权边界、迁移成本、向后兼容。

      FDE 交付的核心难题
      07

      人的角色

      "Humans should move up the stack, not out of the loop."——在合适的时间、合适抽象层做 oversight；许多挑战都需要人的引导。

      Layer 0 的最后守门人
      Trehan & Chopra 六种失败模式

      这是当前 LLM「从想法到论文」最诚实的压力测试（arXiv:2601.03315，45–50 篇种子 / 4 想法进入完整流程 / 只有 1 篇成稿）。六种反复出现的失败模式建议直接抄下来做反面清单：

      1. 训练数据默认偏见：用旧库、过时命令、假设与实际 repo/dataset 无关
      2. 执行压力下的 implementation drift：太复杂时退回更简单常见的方案
      3. 记忆与 context 退化：长周期项目丢关键细节，除非日志作为持久 artifact 落盘
      4. 过度乐观：噪声/失败下也宣告成功（Bubeck 2025 的 "p-hacking and eureka-ing"）
      5. 领域智能不足：缺乏 tacit craft knowledge——预测实现复杂度、判断实验结果合理性
      6. 弱科学品味：实验能跑但答非所问

      八FDE 映射#

      Field Mapping · 对内启示

      1. iHarness 是新的组织资产：不同基座模型进化出不同的 harness（Self-Harness 已实证）——ToB 客户不该问「用哪款模型」，该问「哪套 harness 匹配我的领域约束」。Skill / 状态文件 / eval judge 都是 harness 的物化。
      2. iiLayer 0 × Layer 1 是防 reward hacking 的抽象：SeedAgent × RSI 实验的核心结构（冻结公理/裁判 + 可进化协议）恰好对应 Lilian 提到的「evaluator must live outside the evolution loop」——把这条抽象产品化是 FDE 的机会。
      3. iiiMeta-Harness 应用还未成熟：Level 5「优化优化器」是 2026 年才刚刚上台阶的方向；对企业客户，Level 3–4（workflow + harness code）是当下唯一有稳定收益的选型区间。Meta-level 应作为 R&D 而非交付。
      4. iv负结果落盘是竞争优势：文献偏向成功、LLM 训练数据不平衡——组织内部若能建立「失败 harness 编辑」的持久 archive，就在 pruning 任务搜索空间时拉开身位。这是 DGM 的 archive 思想的组织化对应。
      5. vIntrospection Threshold 是产品叙事的护栏：向客户 sales AGI/RSI 前景时，务必区分「Harness 层的 RSI 已实证」 vs 「完整意义的 RSI 尚未跨过阈值」——不要为可能不会到来的天花板做承诺。
      6. vi验证器是最贵的资产：Chen et al. 2026-07 综述的 verification hierarchy 排序——formal verifier > sandbox check > learned judge > intrinsic self-assessment。观察到的 self-improvement 强度与验证器层级严格相关。FDE 项目里最该早期投资的是评估基座，不是 agent 本身。
      主结论

      Harness 层的 RSI 是当下 6–12 个月的套利窗口——DGM 在固定模型上把 SWE-bench 从 20 拉到 50 说明基座之上还有巨大工程空间；模型直接改权重的 RSI 尚未到来——SIA 是至今最认真的联合优化尝试，但方法论混杂，且理论侧 Introspection Threshold 摆在那里。乐观与警示同时成立，这是 2026 年中 RSI 讨论最诚实的姿态。

      九待验证问题#

      Open Threads · 留给后续
